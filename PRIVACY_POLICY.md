# Devines Privacy Policy

## Purpose

This privacy policy explains how Devines handles information across its public protocol layer, god interfaces, bridge infrastructure, and future runtime systems.

## Core principle

Devines follows a privacy-first model.

User-sensitive information should never be treated as ordinary public data, and must not be stored in raw readable form in public repositories.

## Current architecture context

Devines currently includes:

- public protocol documentation
- a bridge API layer
- Custom GPT god interfaces
- future website interfaces
- GitHub-backed persistence for lawful state and non-sensitive continuity

## What Devines may process

Depending on the interface used, Devines may process:

- user prompts
- god interaction context
- lawful state updates
- distilled non-sensitive summaries
- maintenance metadata
- action request metadata

## What Devines does not allow

Devines does not allow raw readable sensitive user data to be stored in public GitHub repositories.

Sensitive information should be handled with a private-first approach and, where storage is required in future mature implementations, should use strong encryption before persistence.

## Current storage model

At the current stage, Devines may store:

- lawful mutable state
- distilled non-sensitive knowledge
- review-compatible metadata
- private payload metadata
- maintenance outputs

Devines should not store raw readable sensitive interaction content in public persistence.

## Interface note

Devines may be accessed through:

- Custom GPT interfaces
- bridge-connected services
- future website interfaces

These interfaces are intended to connect to the same lawful continuity while preserving privacy boundaries.

## Security note

Secrets such as API keys and backend tokens must never be committed to public repositories and should remain only in secure deployment environments.

## Public protocol vs runtime systems

The public protocol repository exists to document architecture, law, and system direction.

It is not intended to expose private runtime data, protected user data, or hidden operational secrets.

## Future updates

This policy may evolve as Devines expands its runtime, encryption, storage, and interface capabilities.

Updated versions should remain aligned with the privacy-first law of the protocol.

## Contact

For protocol-level privacy concerns, use the public Devines protocol channels associated with the project repository.
